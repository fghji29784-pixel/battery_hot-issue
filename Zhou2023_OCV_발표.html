배경 · Problem Statement
왜 OCV 측정이 오래 걸리는가?

배터리에 전류를 흘리면 내부에 분극(Polarization)이 발생합니다. 전류를 끊어도 이 분극이 사라지는 데 시간이 걸리며, 그 시간 동안 전압이 서서히 변합니다. 이것이 이완(Relaxation) 과정입니다.

😫 기존 방법의 문제
⏱️
증분 OCV 테스트: 각 SOC 포인트마다 3시간 대기 × 36 포인트 = 약 115시간
📉
저전류 측정법: 미약하지만 전류가 흘러 엄밀한 OCV가 아님
🧮
곡선 피팅법: 미지수 많고 수식 복잡, 긴 데이터 필요
⚡
능동 전류 인가법: SOC 자체를 변화시켜 오차 발생
✅ 이 논문의 해결책

이완 초기 몇 분의 전압 데이터만으로,
1차/2차 미분비에서 시정수 τ를 실시간 계산 →
완전 이완 없이도 OCV 예측

↑ tc 시점의 미분값만으로 OCV를 계산 — 이완 완료를 기다릴 필요 없음

핵심 이론 · Core Theory
1차 RC 회로 모델과 시정수 τ
전류 차단 후 이완 (I = 0)
Vbat(t) = OCV + ΔV₀ · e−t/τ
τ = Rp × Cp = 시정수 (분극이 사라지는 속도)
🎯 핵심 아이디어: τ를 미분으로 직접 계산

V(t) = OCV + ΔV₀·e−t/τ 를 두 번 미분하면:

dV/dt = −ΔV₀/τ · e−t/τ
d²V/dt² = ΔV₀/τ² · e−t/τ
두 식을 나누면 ΔV₀ 소거 →
τ = −(dV/dt) / (d²V/dt²)
단, t = tc 시점에서 평가
💡 이것이 혁신적인 이유
✅
ΔV₀, R₀, Rp, Cp 등 어떤 파라미터도 미리 알 필요 없음
✅
측정 전압의 1·2차 미분비만으로 τ 즉시 계산
✅
τ 계산 후: OCV = V(tc) − (dV/dt)|tc × τ
✅
모든 SOH·온도·화학계에 범용 적용 가능

수치 미분: tc 중심 ±90초 데이터에 최소제곱법 적용 (노이즈 저감)

알고리즘 · Critical Time Criterion
언제 계산을 시작하는가? — tc 결정 기준

τ(t)는 이완 초기에 불안정하다가 점점 수렴합니다. "τ가 충분히 안정화된 시점 tc"를 자동으로 결정하는 기준이 필요합니다.

📏 매 초마다 계산하는 지표
ΔV(t) = V(t+1) − V(t)
|avg.ΔV| = |최근 60초 ΔV의 평균|

전압 변화가 작아질수록 τ가 안정화됨 → OCV 예측 가능

🚦 5가지 기준 조건 (논문 Section 3.2)
기준조건특성
C1 ▴|avg.ΔV| < 0.05 mV빠름, 오차 가능
C2 ◆|avg.ΔV| < 0.04 mV중간
C3 ●|avg.ΔV| < 0.03 mV★ 최종 채택 기준
C4 ★|avg.ΔV| < 0.02 mV역효과 가능
C5 ■|avg.ΔV| < 0.025 mV이상 케이스 전용
⚠️ 이상 케이스 감지: k 파라미터
k = (t₂ − t₃) / (t₁ − t₂)
k ≈ 1.3 → 정상 → C3 사용
k > 2 → 이상 → C5 전환
주로 SOC ≈ 50~55% 평탄 구간에서 발생

※ SOC=85%, 25°C, Cell #1 기준 재현. C3 채택 시 오차 ≈ 0 (영역 R 안에 위치)

🔄 CTR vs DTR: 왜 차이가 나는가?

CTR: 충전 후 OCV로 자연 하강 → 수렴 빠름 → 더 짧은 시간, 더 정확
DTR: 방전 후 OCV로 상승 → 구동력 약함 → 수렴 느림

📍 SOC ≈ 50~55% 구간 주의

💻 구현 계획 (Python)
# τ 실시간 계산 핵심 로직
dVdt = polyfit(t_win, V_win, 1)[0]
d2Vdt = polyfit(t_win, V_win, 2)[0]*2
tau = −dVdt / d2Vdt
OCV = V_tc − dVdt * tau
